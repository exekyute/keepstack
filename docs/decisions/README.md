# Architecture decision records

Short notes on the choices that shaped Keepstack: the context, the decision, and
the honest tradeoff. They are numbered in the order the decisions were made, so
reading them top to bottom is a tour of how the design came together.

- [ADR-0001: SQLite over a database server](ADR-0001-sqlite-over-a-database-server.md)
- [ADR-0002: Content-addressable storage](ADR-0002-content-addressable-storage.md)
- [ADR-0003: A zero-build frontend](ADR-0003-zero-build-frontend.md)
- [ADR-0004: Optional, offline-capable AI](ADR-0004-optional-offline-ai.md)
- [ADR-0005: Open standards as core, not an add-on](ADR-0005-standards-as-core.md)

Each record follows the same small shape: Context, Decision, Consequences (the
good and the cost). New decisions get a new numbered file rather than editing an
old one, so the reasoning stays a history.
